# GTMSA v1.1.0 — Bug Fix Design

**Date:** 2026-03-15
**Plugin:** `geotapp-multilingual-seo-automation`
**Approach:** A — chirurgico, minimo indispensabile
**Version bump:** 1.0.3 → 1.1.0

---

## Obiettivo

Correggere tre bug che impediscono il funzionamento corretto con le lingue target:
**it** (sorgente), **en, de, fr, es, pt, nb, da, sv, ru** (target).

---

## Fix 1 — DeepL source/target language maps separati

### Problema
`gtmsa_map_deepl_language()` è usata sia per `source_lang` che per `target_lang`.
DeepL API v2 accetta `EN` come source ma **richiede** `EN-GB` (o `EN-US`) come target.
Usando `EN` come target si ottiene errore 400 o comportamento deprecato → le traduzioni in inglese non vengono prodotte.

### Soluzione
Sostituire `gtmsa_map_deepl_language()` con due funzioni:

```php
gtmsa_map_deepl_source($lang): ?string
// EN, PT, NB, DE, FR, ES, IT, NL, SV, DA, RU

gtmsa_map_deepl_target($lang): ?string
// en → EN-GB, pt → PT-PT, resto invariato rispetto a source
```

`gtmsa_translate_text_batch()` chiamerà `gtmsa_map_deepl_source()` per `$source_language`
e `gtmsa_map_deepl_target()` per `$target_language`.

### Mapping target confermato
| Lang code | DeepL target |
|-----------|-------------|
| it | IT |
| en | EN-GB |
| de | DE |
| fr | FR |
| es | ES |
| pt | PT-PT |
| nl | NL |
| sv | SV |
| da | DA |
| nb | NB |
| ru | RU |

## Fix 2 — Controllo lingue registrate in Polylang

### Problema
`gtmsa_translate_one_post()` itera su `$target_languages` senza verificare che
le lingue siano effettivamente registrate in Polylang.
Chiamare `pll_set_post_language($id, 'nb')` su una lingua non registrata fallisce
silenziosamente, producendo post orfani senza lingua.

### Soluzione
All'inizio di `gtmsa_translate_one_post()`, dopo aver ottenuto `$target_languages`,
recuperare le lingue attive in Polylang:

```php
$registered = pll_languages_list(['fields' => 'slug']);
$target_languages = array_values(array_intersect($target_languages, $registered));
```

Se una lingua viene filtrata, loggare:
```
error_log('GTMSA: skipping lang $lang — not registered in Polylang');
```

## Fix 3 — Limite retry sulla coda

### Problema
Post che falliscono la traduzione vengono rimessi in coda indefinitamente.
Un articolo problematico (es. contenuto malformato, limite DeepL raggiunto)
blocca la coda senza mai risolversi.

### Soluzione
Cambiare la struttura interna della queue da `int[]` a `array<{id: int, retries: int}>`.

**Migrazione trasparente:** `gtmsa_get_queue()` rileva le entry in formato vecchio (int)
e le converte automaticamente a `['id' => $int, 'retries' => 0]`.

**Max retry:** 5. Al sesto fallimento il post viene droppato con:
```
error_log('GTMSA: post $id dropped from queue after 5 retries');
```

Le funzioni pubbliche `gtmsa_enqueue_post_id()` e `gtmsa_process_translation_queue()`
restano invariate nell'interfaccia esterna; la struttura interna è opaca.
